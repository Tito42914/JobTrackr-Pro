## Work Completed

### Refined Form Layout

Updated:

```text
src/features/applications/components/ApplicationForm.tsx
```

Changes:

- Moved the `notes` textarea outside the two-column grid.
- Moved the submit button outside the two-column grid.
- Kept the first six fields in the grid:
  - Company.
  - Role.
  - Location.
  - Date applied.
  - Status.
  - Job link.

Result:

- `notes` now spans the full form width.
- The submit button now sits on its own row.
- The form layout is cleaner and easier to scan.

### Added Initial Values Support

Updated:

```text
src/features/applications/components/ApplicationForm.tsx
```

Changes:

- Imported `JobApplicationInput`.
- Added an optional `initialValues` prop.
- Used `defaultValue` on form fields.
- Used the existing starter application values when `initialValues` are provided.
- Kept create mode blank by default.

Learning points:

- `initialValues?: JobApplicationInput` means the prop is optional.
- `defaultValue` gives fields a starting value without making them controlled inputs.
- This is still not a `useState` form.
- This is a bridge between static forms and fully interactive forms.

### Prefilled Edit Form

Updated:

```text
src/app/applications/[id]/edit/page.tsx
```

Changes:

- Passed the matched starter application's editable fields into `ApplicationForm`.
- Did not pass generated fields such as:
  - `id`
  - `createdAt`
  - `updatedAt`

Result:

- `/applications/[id]/edit` now shows a prefilled form for the selected starter application.
- `/applications/new` still shows a blank form.
