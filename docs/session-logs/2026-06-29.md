## Work Completed

### Created `useApplications`

Created:

```text
src/features/applications/hooks/useApplications.ts
```

The hook currently provides:

- `applications`
- `addApplication`
- `updateApplication`
- `getApplicationById`

Learning points:

- The hook owns the application array in React state.
- It starts from `starterApplications`.
- `addApplication` converts `JobApplicationInput` into a full `JobApplication`.
- The hook adds generated fields:
  - `id`
  - `createdAt`
  - `updatedAt`
- This is still in-memory only.

### Created `ApplicationsProvider`

Created:

```text
src/features/applications/providers/ApplicationsProvider.tsx
```

The provider:

- Calls `useApplications`.
- Stores the hook result in React context.
- Exposes `useApplicationsContext`.

Learning points:

- Calling `useApplications` separately in each page would create separate state.
- A provider lets the `/applications` route tree share one state instance.
- The context helper protects against using the hook outside the provider.

### Wrapped The Applications Route Segment

Created:

```text
src/app/applications/layout.tsx
```

This layout wraps all application routes in `ApplicationsProvider`.

Covered routes:

```text
/applications
/applications/new
/applications/[id]
/applications/[id]/edit
```

Learning point:

- App Router layouts can wrap a route segment and its children.
- A Server Component layout can render a Client Component provider.

src/features/applications/components/ApplicationsList.tsx

The list now:

- Reads `applications` from context.

Updated:

```text
src/app/applications/page.tsx
```

The page now renders `ApplicationsList` instead of importing `starterApplications` directly.

Learning point:

- The page handles page layout and copy.
- The list component handles application rendering.
- The provider/hook owns data state.

### Created `NewApplicationForm`

Created:

```text
src/features/applications/components/NewApplicationForm.tsx
```

The component:

- Uses `useApplicationsContext`.
- Uses `useRouter`.
- Calls `addApplication` on submit.
- Navigates back to `/applications`.

Updated:

```text
src/app/applications/new/page.tsx
```

The new route now renders `NewApplicationForm`.

Learning point:

- `ApplicationForm` remains reusable.
- `NewApplicationForm` decides what create submission means.
- This keeps route-specific behaviour out of the generic form component.

## What Works Now

- `/applications` reads from shared in-memory application state.
- `/applications/new` submits into that shared state.
- After submitting a new application, the user is routed back to `/applications`.
- The new application should appear in the list until the page is refreshed.
