## Work Completed

### Styled `StatusBadge`

Updated:

```text
src/features/applications/components/StatusBadge.tsx
```

Changes:

- Added Tailwind styling for a pill-shaped status badge.
- Added status-specific colour styles.
- Kept the status style map typed with `Record<ApplicationStatus, string>`.
- Continued using `APPLICATION_STATUS_LABELS` for display text.

Learning points:

- `Record<ApplicationStatus, string>` can be used for more than labels.
- It can also guarantee that every valid status has a matching style.
- Styling can stay type-safe when it depends on typed data values.

### Styled `ApplicationCard`

Updated:

```text
src/features/applications/components/ApplicationCard.tsx
```

Changes:

- Added a card shell with border, background, padding, rounded corners, and shadow.
- Added hover polish.
- Placed the role/company section and status badge in a responsive flex layout.
- Displayed location and applied date in a small responsive metadata grid.

Learning points:

- The component still receives one `JobApplication`.
- Styling did not change the data contract.
- `ApplicationCard` still composes `StatusBadge`.
- Tailwind classes are being added directly to TSX elements.

### Styled the `/applications` Page

Updated:

```text
src/app/applications/page.tsx
```

Changes:

- Added a full-page `main` shell.
- Added a light slate background.
- Centered content with a max width.
- Added a page header with label, title, and description.
- Added spacing between application cards.

Learning points:

- App Router pages can act as page-level composition files.
- Feature components handle repeated UI.
- Page files can provide the surrounding layout and copy.
