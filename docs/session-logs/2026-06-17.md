## StatusBadge Component

Created:

```text
src/features/applications/components/StatusBadge.tsx
```

Purpose:

- Display a human-friendly label for an application status.
- Keep lowercase internal status values separate from user-facing text.

Key idea:

```text
status value -> status label
```

Example:

```text
applied -> Applied
interviewing -> Interviewing
```

Learning point:

- The component receives a typed `status` prop.
- The prop must be an `ApplicationStatus`.
- The component uses `APPLICATION_STATUS_LABELS[status]` to find the display label.

## ApplicationCard Component

Created:

```text
src/features/applications/components/ApplicationCard.tsx
```

Purpose:

- Display one complete job application record.
- Compose the `StatusBadge` component inside the card.

Displayed fields:

- Role.
- Company.
- Location.
- Date applied.
- Status label.

Learning point:

- The component receives an `application` prop.
- The prop is typed as `JobApplication`.
- TypeScript can now verify that fields such as `role`, `company`, `location`, `dateApplied`, and `status` exist.
- `ApplicationCard` passes only the needed `status` value down to `StatusBadge`.

## Applications Page Updated

Updated:

```text
src/app/applications/page.tsx
```

The page now:

- Imports `starterApplications`.
- Imports `ApplicationCard`.
- Maps over the starter data.
- Renders one `ApplicationCard` per starter application.

Data flow achieved:

```text
starterApplications
  -> ApplicationsPage
  -> ApplicationCard
  -> StatusBadge
```

This is the first real typed data-to-UI chain in JobTrackr Pro.
