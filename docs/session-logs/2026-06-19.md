## Work Completed

### Linked Application Cards to Dynamic Routes

Updated:

```text
src/features/applications/components/ApplicationCard.tsx
```

Changes:

- Imported `Link` from `next/link`.
- Wrapped the application role in a `Link`.
- Built the link URL from the typed application ID.

Example route generated by the card:

```text
/applications/starter-1
```

Learning points:

- `Link` enables client-side navigation in Next.js.
- The card already receives a full `JobApplication`, so it can use `application.id` to build the correct detail URL.
- This connects the reusable card component to the App Router route structure without adding state or localStorage.

### Built the Dynamic Application Details Page

Updated:

```text
src/app/applications/[id]/page.tsx
```

Changes:

- Read the dynamic route parameter from `params`.
- Used the `id` route parameter to find a matching starter application.
- Rendered a styled detail page for the matching application.
- Reused `StatusBadge` on the details page.
- Added a back link to `/applications`.
- Used `notFound()` when no matching starter application exists.

Learning points:

- The `[id]` folder represents a dynamic URL segment.
- A URL like `/applications/starter-1` provides an `id` value of `starter-1`.
- The page can use that ID to find the matching record.
- `notFound()` is the Next.js way to trigger a 404-style result for missing records.

## Result

The app now supports:

- Viewing the styled applications list.
- Clicking an application role.
- Navigating to `/applications/[id]`.
- Rendering details for the selected starter application.
- Handling unknown IDs with `notFound()`.

This is the first complete list-to-detail flow in JobTrackr Pro.
