## App Router Learning

- Learned that Next.js App Router uses the filesystem to define URLs.
- Established the core routing rules:
  - Each folder under `src/app` creates a URL segment.
  - A route becomes publicly accessible when that folder contains a `page.tsx` file.
  - `src/app/page.tsx` maps to `/`.
  - Nested folders create nested URL paths.
  - A folder wrapped in square brackets, such as `[id]`, creates a dynamic route segment.
- Clarified that `[id]` uses square brackets and is not an array.
- Learned how a dynamic URL value maps to a route parameter:

  ```text
  /applications/123
                |
            id = "123"
  ```

- Clarified that the ID will later be used to find the matching application record.
- Confirmed that static and dynamic routes can coexist:
  - `/applications/new` uses the fixed `new` route.
  - `/applications/123` uses the dynamic `[id]` route.
