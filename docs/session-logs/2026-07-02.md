## Work Completed

### Added delete behaviour to the data layer

Updated:

```text
src/features/applications/hooks/useApplications.ts
```

Added:

```text
deleteApplication
```

The delete helper:

- Takes an application ID.
- Filters that application out of the saved array.
- Updates React state.
- Allows the existing localStorage effect to persist the updated list.

Learning point:

- Delete is another state transition in the same data layer as create and update.
- Because localStorage saving is already tied to the applications array, deleting from state also deletes from localStorage.

### Added delete UI to application details

Updated:

```text
src/features/applications/components/ApplicationDetails.tsx
```

The detail page now:

- Gets `deleteApplication` from context.
- Uses `window.confirm` before deleting.
- Deletes the selected application.
- Redirects back to `/applications`.
- Uses an `isDeleting` transition state to avoid showing a 404 during the redirect.

Learning point:

- Deleting from a detail page removes the very data that page is currently rendering.
- Without a transition guard, the page can briefly re-render as "not found" before navigation completes.
- `isDeleting` tells the component that the missing application is expected during the delete flow.

### Added search and status filtering

Updated:

```text
src/features/applications/components/ApplicationsList.tsx
```

The list now includes:

- Search input.
- Status dropdown.
- Search across company, role, and location.
- Status filtering using shared status constants.
- A no-matches empty state.

Learning point:

- `applications` is the source data from shared state.
- `filteredApplications` is a temporary display-only version.
- Filtering does not mutate saved data or localStorage.
- The UI maps over `filteredApplications` because that is the list the user currently wants to see.
